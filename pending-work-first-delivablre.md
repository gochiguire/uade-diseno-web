Para tu proyecto del Trabajo Práctico Nro. 1 “Identikit”, debes integrar una serie de componentes obligatorios y requisitos técnicos que reflejen la evolución cronológica de la maquetación web
.
A continuación, detallo las cosas que tenés que usar y aplicar en tu proyecto:
1. Componentes Obligatorios del Sitio
El trabajo debe estar dividido en las siguientes etapas o secciones:
Página de bienvenida: Una sección de inicio o "Sobre mí" donde te presentas y compartes tus intereses
.
Implementación de Tablas: Al menos una sección o página que utilice la etiqueta <table> para organizar información (visto como tecnología de la Clase 2)
.
Menú de navegación con Listas: Un menú realizado con listas (<ul> y <li>) que permita navegar por todas las secciones del sitio
.
Trabajo creativo a elección: Puede ser un juego, una pieza de arte, diseño, o una trivia interactiva donde el usuario navega a través de diferentes páginas HTML según sus respuestas
.
Contenido multimedia (Playlist): Una página con música o videos favoritos, utilizando posicionamiento estático (position: static) para su maquetación
.
2. Requisitos Técnicos de HTML y CSS
Tu código debe demostrar el uso correcto de las herramientas vistas en clase:
Estructura Semántica: Uso de etiquetas como <header>, <main>, <section>, <article> y <footer>
.
Modelo de Cajas (Box Model): Aplicación de propiedades como margin (espacio exterior), padding (espacio interior), border y box-sizing: border-box para controlar el tamaño de los elementos
.
Tipografía Personalizada: Debes importar al menos dos o tres fuentes distintas desde Google Fonts usando la regla @import en tu archivo CSS
.
Propiedades de Visualización (Display): Uso de block, inline, e inline-block para estructurar elementos (especialmente para que el menú de navegación sea horizontal)
.
Estilos y Color: Definición de una paleta de colores coherente (usando valores RGB o nombres) y aplicación de armonías cromáticas
.
3. Interactividad y Recursos Visuales
Efectos Hover: Uso de la pseudoclase :hover para que elementos como botones o links cambien de color o apariencia (por ejemplo, ajustando la opacity) al pasar el mouse
.
Imágenes y Favicon: Uso de imágenes con formatos adecuados (JPG, PNG) y la inclusión obligatoria de un favicon (ícono en la pestaña del navegador) mediante la etiqueta <link rel="icon"...>
.
Enlaces de Ancla: Implementación de etiquetas <a> para navegación interna y externa
.
4. Organización y Publicación
Estructura de Archivos: Organizar el proyecto en subcarpetas (por ejemplo, una carpeta para imágenes y otra para la trivia) y nombrar todos los archivos en minúsculas sin espacios
.
GitHub y GitHub Pages: El proyecto debe subirse a un repositorio en GitHub y estar publicado online a través de GitHub Pages para que sea navegable mediante una URL real
.
Es fundamental que intentes centralizar tus estilos en un solo archivo CSS externo (styles.css) en lugar de usar estilos en línea, para mantener el código ordenado y facilitar cambios en todo el sitio
